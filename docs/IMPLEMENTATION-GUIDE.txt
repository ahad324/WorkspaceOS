# IMPORTANT

This document is the single source of truth for implementing WorkspaceOS.

Every AI agent, contributor, or developer working on this project MUST
read this document before writing code.

If there is a conflict between implementation and architecture, the
Software Architecture Documents (SAD) and Architecture Decision Records
(ADR) take precedence.

WorkspaceOS SHALL NEVER sacrifice architecture, security, correctness,
or maintainability for short-term convenience.

This guide defines WHAT to build, WHEN to build it, and HOW to build it.

--------------------------------------------------------------------

# 1. Project Mission

Build the world's fastest, most secure, and most intelligent local
workspace runtime for AI.

WorkspaceOS should allow AI assistants to understand codebases deeply,
perform authorized actions, and maintain complete repository awareness
while remaining lightweight enough to run efficiently even on low-end
hardware.

Primary goals:

• Near-zero idle CPU
• Low memory usage
• Extremely fast startup
• Incremental repository indexing
• Intelligent context generation
• Provider-independent MCP implementation
• Strong security boundaries
• Modular architecture
• Long-term maintainability

--------------------------------------------------------------------

# 2. Non-Negotiable Principles

Every implementation MUST satisfy:

✓ Security First
✓ Correctness First
✓ Performance Through Good Design
✓ Event-Driven Architecture
✓ Incremental Computation
✓ Deterministic Behavior
✓ Modular Design
✓ Clean Architecture
✓ Testability
✓ Benchmarkability

Never optimize based on assumptions.

Measure first.

Optimize second.

--------------------------------------------------------------------

Metadata extraction

Symbol extraction

Repository graph generation

Exports

Relationships

# 9. Phase 5

Search Engine

Dependency search

Architecture search
