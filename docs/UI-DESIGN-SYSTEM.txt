# PURPOSE

This document defines the complete visual language for WorkspaceOS.

Every screen, component, color, animation, spacing rule, typography rule
and interaction MUST follow this document.

The goal is to build a desktop application that looks modern,
professional, premium and timeless rather than trendy.

The UI should feel like software built by a world-class product team.

====================================================================
DESIGN PHILOSOPHY
====================================================================

Keywords

• Modern
• Minimal
• Premium
• Fast
• Clean
• Technical
• Calm
• Confident
• Information Dense
• Developer Focused

The interface should communicate precision rather than decoration.

Avoid visual noise.

Every pixel should have a purpose.

Whitespace is preferred over unnecessary borders.

====================================================================
DESIGN REFERENCES
====================================================================

The design language should be inspired by the principles (not copied
assets) of:

• Linear
• Raycast
• Arc Browser
• GitHub Desktop
• Warp Terminal
• Notion
• VS Code
• Tailscale
• Stripe Dashboard

====================================================================
THEME
====================================================================

Dark Mode is the primary design target.

Light Mode should also be supported using the same design tokens.

ANIMATIONS
====================================================================

Animations should feel fast.

120–220ms

====================================================================

====================================================================

EMPTY STATES
====================================================================

Every empty page should explain:

====================================================================
ACCESSIBILITY
====================================================================

WCAG AA contrast minimum.

Keyboard navigation.

Visible focus states.

Screen reader labels where appropriate.

Never rely on color alone.

====================================================================
COMPONENTS
====================================================================

shadcn/ui

Lucide React

Motion

React Hook Form

Zod

====================================================================
WINDOW STYLE
====================================================================

Native Tauri window.

Custom title bar.

Rounded corners (where supported).

Native shadows.

Blur effects should be subtle and used sparingly.

====================================================================
PERFORMANCE
====================================================================

Avoid unnecessary re-renders.

Lazy load large pages.

Virtualize long lists if required.

Keep animations GPU-friendly.

====================================================================
DO NOT
====================================================================

Do not use:

Glassmorphism everywhere.

Heavy gradients.

Neon effects.

Oversized shadows.

Random bright colors.

Comic fonts.

Material Design defaults.

Bootstrap styling.

Generic admin dashboard templates.

====================================================================
GOAL
====================================================================

When users open WorkspaceOS they should immediately think:

"This feels fast."

"This feels premium."

"This feels like a professional developer tool."

The interface should emphasize clarity, speed and confidence over visual
flashiness.
